# 离线摆摊点单 App 需求与技术说明（v1.0）

## 1. 产品目标

面向摆摊场景，提供一个完全离线、打开即用的安卓点单计价 App。核心目标：

- 快速选商品并实时计算总价
- 操作路径短（核心动作不超过 2 步）
- 弱网络/无网络可持续使用
- 重启后商品与配置不丢失

## 2. 用户角色

- 员工（默认角色）
- 无登录、无权限分级，点开即用

## 3. 范围与边界

### 3.1 本期范围（必须）

- 点单主页：分类、搜索、商品网格、实时总价、完成清空
- 商品管理：新增/编辑/删除/上下架、图片录入
- 已选清单：加减改数量、小计、自动移除
- 本地持久化：商品、分类、图片路径、设置
- 完全离线运行

### 3.2 可选范围（建议）

- 拼音首字母搜索
- 商品导入/导出（JSON 或 CSV）
- 完成前二次确认
- 未完成订单恢复开关

### 3.3 非目标（本期不做）

- 在线接口、云同步
- 多门店账户体系
- 打印小票

## 4. 信息架构（页面清单）

1. 点单主页（核心）
2. 已选清单弹层/页面
3. 商品管理页
4. 新增/编辑商品页
5. 分类管理页（可选）
6. 设置页（可选）

## 5. 功能需求

## 5.1 商品管理

### 5.1.1 商品字段

- `id`：Long，主键，自增（不在 UI 展示）
- `name`：String，必填，默认可重复（设置中可开启唯一校验）
- `price`：Decimal(10,2)，必填，默认要求 `> 0`（可在设置允许 `=0` 赠品）
- `imageUri`：String，可空，本地私有目录路径或 content uri
- `status`：Enum，`ON_SALE` / `SOLD_OUT`
- `categoryId`：Long，可空
- `weight`：Int，默认 0，用于自定义排序
- `createdAt` / `updatedAt`：Long（时间戳）

### 5.1.2 新增/编辑流程

1. 进入商品管理
2. 点击新增或编辑
3. 填写名称、价格、分类、状态、权重
4. 添加图片：相册选择（优先 Photo Picker）或拍照
5. 图片处理：推荐 1:1 裁剪、压缩后存 App 私有目录
6. 保存成功后，点单页立即可见（列表自动刷新）

### 5.1.3 删除与上下架

- 删除：支持单个删除，删除前二次确认
- 下架：商品仍保留在管理页，但点单页默认不显示（可设置显示）

### 5.1.4 导入导出（可选）

- 导出：商品与分类导出为 JSON/CSV
- 导入：选择文件后预检字段，失败行给出原因

## 5.2 点单计价（主页面）

### 5.2.1 布局结构

- 顶部：摆摊名（可选）+ 搜索框 + 分类栏（Tab/横向列表）
- 中部：商品网格（图片 + 名称 + 价格 + 已选徽标）
- 底部吸底栏：总价（实时）+ 完成按钮

### 5.2.2 商品筛选与排序

至少实现以下 3 类能力：

- 分类过滤：按分类显示
- 搜索过滤：输入关键字实时过滤（名称包含）
- 自定义排序：按权重排序（权重高优先，或在设置切换）

可选：支持拼音首字母搜索。

### 5.2.3 商品卡片交互

默认采用模式 A（摆摊高频最优）：

- 单击商品卡片：数量 `+1`
- 右上角徽标显示已选数量（仅数量 > 0 显示）
- 长按商品卡片：弹出操作（改数量 / 清零 / 备注可选）

交互要求：

- 点击反馈动画轻量（缩放或波纹）
- 快速连点不丢事件（10 次连点应精确 `+10`）

## 5.3 已选商品清单

### 5.3.1 入口

- 点击底部总价区域打开（BottomSheet 或新页面）

### 5.3.2 列表项字段

- 商品图（可选）
- 名称
- 单价
- 数量（可编辑）
- 小计（`price * qty`）
- `+` / `-` 按钮

### 5.3.3 数量编辑规则

- 数量必须为整数且 `>= 0`
- 改为 `0`：从已选清单移除
- 允许大数量（如 999）；可配置上限提示（可选）
- 点击数量弹数值输入框，使用数字键盘

## 5.4 总价与完成

### 5.4.1 总价规则

- `total = Σ(item.price * item.qty)`
- 任一数量变化后实时刷新
- 显示格式：`¥`（可选）+ 两位小数（统一 `0.00`）

### 5.4.2 完成按钮

点击后执行：

- 清空本单已选商品
- 总价归零
- 返回主商品列表状态

可选：开启“完成前确认弹窗”。

## 5.5 边界与异常

- 无图商品：显示统一占位图，不影响点击热区
- 非法价格：禁止保存并提示（非数字、负数）
- 非法数量：禁止提交并提示（小数、负数、空）
- 商品被下架但在购物车中：保留已选项直到完成/清空
- 异常退出恢复策略：
- 默认不恢复未完成订单（简洁）
- 设置可开启“恢复上次未完成订单”

## 6. 本地数据与离线设计

## 6.1 存储方案

- 数据库：Room（SQLite）
- 图片：App 私有目录（`filesDir/images` 或 `cacheDir/images`）
- 数据库存路径：`imageUri`
- 设置项：DataStore（Preferences）

## 6.2 推荐表结构

- `products(id, name, price, image_uri, status, category_id, weight, created_at, updated_at)`
- `categories(id, name, weight, created_at)`
- `cart_items(product_id, qty, updated_at)`（若启用恢复）
- `settings(key, value)` 或 DataStore

索引建议：

- `products(name)`（搜索）
- `products(category_id, status, weight)`（列表查询）

## 6.3 离线保证

- 所有核心流程不依赖网络
- 无网络时功能等价可用
- 重启后商品、分类、设置可恢复

## 7. 技术实现建议（Android）

- 语言：Kotlin
- UI：Jetpack Compose（或 RecyclerView 方案）
- 架构：MVVM + Repository
- 数据：Room + DataStore
- 图片加载：Coil
- 并发：Kotlin Coroutines + Flow

关键实现点：

- 商品列表使用分页/惰性网格（`LazyVerticalGrid`）
- 列表项 key 稳定，减少重组
- 点击加数量走内存态 + 持久化节流（避免频繁磁盘写入）
- 价格统一用 `BigDecimal` 或“分”为单位 `Long` 避免浮点误差

## 8. 权限与系统能力

- 相册选图：优先 Android Photo Picker，尽量免存储权限
- 拍照：需要相机权限
- 图片统一写入 App 私有目录，避免外部存储兼容问题

## 9. 非功能需求

### 9.1 性能

- 商品 200 到 1000 条下滚动流畅
- 连续点击稳定，不丢计数
- 图片加载与滚动无明显卡顿

### 9.2 易用性

- 大字号、大按钮，适合站立操作
- 总价与完成按钮高对比突出
- 核心操作不超过 2 步

### 9.3 稳定性

- 异常退出后商品数据不丢失
- 崩溃恢复后数据库一致

## 10. 验收标准（测试用例）

- TC01：新增商品（有图/无图）后，主页可见且可点击
- TC02：分类切换正确，商品列表与数量显示正常
- TC03：搜索关键字过滤正确，清空后恢复完整列表
- TC04：点击商品数量递增正确，徽标实时更新
- TC05：已选清单中 `+/-` 可用，减至 0 自动移除
- TC06：点击数量可直接输入，非法输入有提示且不落库
- TC07：总价计算实时且精确到两位小数
- TC08：点击完成后购物车清空，总价归零
- TC09：飞行模式/无网络下功能完整可用
- TC10：重启 App 后商品与分类数据仍存在

- M1（1-2 天）：项目骨架、数据库、商品管理 CRUD
